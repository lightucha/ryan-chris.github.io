초짜의 Git Pull-Request(PR)

2018-08-09
Git
PR, Pull-Request, git

협업 해보고자 PR 자료를 찾으시는 분들에게 도움이 되길 바랍니다.

사실, 저도 아직도 github을 잘 모르겠습니다만
처음 github pull request를 하면서 겪었던 시행착오를 적어보겠습니다.

들어가기 전에

  • github 계정이 있음을 전제로 합니다 :-)
  • git으로 add, commit, push 해본 경험이 있으시길 바랍니다.

PR이 뭔가요?

github에서 pull request를 PR이라 부르시더군요..

PR을 하는 이유는 여러가지가 있겠지만, 가장 주된 목적은 협.업. 이지 않을까 싶습니다.
저도 PR을 시도한 이유는 수많은 최종 버전 파일을 만들지 않고, 멋있게(?) 프로젝트를 해보고 싶어서..입니다.

도전! 이번엔 실전이다.

이제 본격적으로 pull request를 해보겠습니다.
많은 구글링을 통해서 제일 간단한(?) 방법이라고 생각되는 절차들을 정리했습니다.
git을 사용하는 정석이 아닐지도 모릅니다만, 실행은 되는 것 같기에 공유합니다.

  1. Fork
  2. Clone 및 git remote 설정
  3. branch 생성
  4. (나만의 작업 후) git add, commit, push
  5. git pull request 생성
  6. Merge pull request 대기
  7. Local merge 후 branch 삭제

1. Fork 하기

같이 작업 할 저장소로 이동하여 콬! Fork 해옵니다.

저는 홀로 연습을 위해 Organization을 만들었습니다.
혼자 연습하시는 분들은 한 번 해보셔도 좋을 듯 합니다.

2. Clone 하고 remote 설정하기

Fork가 되면 개인 저장소에 원본의 프로젝트가 생깁니다.

(참고)Clone하게 되면 자동으로 Origin remote로 설정됩니다!!

그리고 나서 해야 할 일은 원본 저장소를 remote에 등록하는 것!!

현재 remote 상태를 보는 명령어는?!

3. 작업을 위한 branch 생성하기

보통 혼자 작업을 하게 되면 master branch로 하게 되는 것 같습니다.
branch를 만들어 관리하는 이유는 코드를 통째로 복사한다고 생각하시면 좋을 것 같습니다.

현재 local에 존재하는 branch를 확인합니다.

새로운 작업을 위한 branch를 생성하겠습니다.


branch가 생성이 되었으니 원본과 무관하게 독립적으로 작업을 할 수 있습니다.

4. 작업 하고 git add, commit, push 하기

그렇다면 이제 원본 저장소에서 가져온 것을 수정(개발) 한 후, 반영을 해보겠습니다.
먼저, Fork 해 온 prtest의 저장소에는 README.md 파일만 있었습니다.
README.md 파일을 수정해 보고, test1.html이라는 파일을 만들어 반영하겠습니다.

혹시라도 모르시는 분들을 위해,

현재 Local과 Stage의 상태를 확인합니다.

수정 된 코드를 Stage로 올립니다.





Push하기 위해 commit을 합니다.

이제 저장소로 보냅니다.





5. Github에서 Pull Request 생성하기

push를 완료하고 나면, Compare & Pull Request가 활성화 됩니다.

주의하실 것은 좌측에 Branch란이 master로 되어 있음을 인지하셔야 합니다!!
위에서 작업한 Branch가 아니란 말이죠!! 바꿔주셔야 합니다!!


제가 작업한 것들이 생겼쥬??

이제 Compare & Pull Request 콬! 클릭하시고, 무엇을 수정한 코드인지 설명을 써줍니다.

6. Merge 대기 및 코드 리뷰

PR을 생성하고 나면, 내가 수정한 혹은 개발한 코드를 관련 팀원들과 리뷰를 할 수 있습니다.




이후 관리자가 Accept 해준다는 의미로 내가 수정한 코드를 원본 저장소로 merge 해줍니다.
그러면 아래와 같이 prtest 저장소에 짜~안! 반영이 되었습니다.

7. Merge 이후 Local 반영(동기화) 및 사용했던 branch 삭제하기

Merge가 된 것을 확인하고 나면, 원본 저장소와 개인의 Local과 동일하게 만들어 주어야 합니다.
동기화 하기 위해 pull 해줍니다.





그리고 merge된 branch는 삭제 해주면 끝.






이후에 반복되는 작업들은 3번부터 반복하시면 됩니다.
정석이 아닐수도 있지만, 일단 PR에 성공했으니 함께 공유합니다.
git도 너무나 공부할게 많아서 쉽지 않지만, 저와 같은 초짜들에게 도움이 되길 바랍니다.
